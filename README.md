# B 站动态评论批量点赞助手

一个用于 B 站动态页面评论区的油猴脚本。脚本通过扫描页面中的评论点赞按钮，自动识别已点赞和未点赞状态，并按顺序尝试为当前已加载的评论点赞。

本项目是我的第一个开源项目，并且主要使用GPT5.5制作。

## 核心功能

### 自动扫描评论点赞按钮

脚本会在 B 站动态页面中扫描评论区的点赞按钮，并尝试识别新版评论区中位于 Shadow DOM 内部的按钮结构。

由于新版评论区使用了 Web Components 和 Shadow DOM，普通的 `document.querySelectorAll()` 很可能无法直接找到评论按钮。本脚本会递归扫描 open Shadow DOM，以提高识别成功率。

### 识别已点赞和未点赞状态

脚本会尽量判断一个评论按钮是否已经处于已点赞状态，避免重复点击导致取消点赞。

当前判断方式包括：

- 按钮或图标颜色是否接近已点赞蓝色 `#00AEEC`
- 图标名称是否包含已点赞相关标识
- 按钮类名是否包含 active / liked 等状态
- aria 或 title 中是否存在已点赞相关信息
- 点赞数是否发生变化

### 防止二次点击取消点赞

点赞按钮通常是开关按钮，同一个按钮点击两次会从“点赞”变成“取消点赞”。

因此脚本会记录已经尝试点击过的按钮，在当前页面生命周期内不会重复点击同一个按钮，从而降低误取消点赞的概率。

### 小步滚动与回扫

为了减少“滚动过头导致漏掉评论”的情况，脚本采用小步滚动和视口回扫的方式：

- 先扫描当前已加载的评论按钮
- 按页面位置从上到下排序
- 处理当前可见区域和已加载区域中的未点赞按钮
- 小步向下滚动
- 滚动后再次扫描附近评论

### 可拖拽悬浮面板

脚本会在页面右下角显示一个悬浮控制面板，包含：

- 开始点赞评论
- 停止
- 已点赞数量
- 候选按钮数量
- 扫描轮次
- 空轮次数

面板支持拖拽，并会保存当前位置。

## 🛠️ 安装指南 (Installation)

1. **安装浏览器插件管理器**：
   * 推荐安装 [Tampermonkey (油猴)](https://www.tampermonkey.net/) 插件（支持 Chrome / Edge / Firefox / Safari）。
2. **安装本脚本**：
   * 在 Tampermonkey 中点击 **“添加新脚本”**。
   * 将本仓库中的脚本源码（`bili-danmaku-assistant.js`）全选复制并粘贴进去。
   * 点击 **文件 -> 保存** (或 `Ctrl+S`)。
3. **打开 B 站动态页面**

   * 打开任意 B 站动态详情页，例如：

```text
https://t.bilibili.com/动态ID
```

   * 页面加载完成后，右下角应该会出现“评论点赞助手”面板。

 **开始使用**

   * 点击面板中的“开始点赞评论”按钮，脚本会开始扫描当前页面中的评论点赞按钮。

   * 如果需要中断，点击“停止”。

## 使用建议

首次使用时，建议先在评论数量较少的动态页面测试，确认脚本能够正常识别点赞状态。

如果页面安装了其他修改 B 站样式或评论区结构的插件，可能会影响按钮颜色、布局或 DOM 结构，从而影响脚本识别效果。

如果发现面板位置异常，可以刷新页面，或在浏览器控制台中执行脚本提供的重置面板位置方法。

## 常见问题

### 为什么脚本没有找到候选按钮？

可能原因包括：

- 评论区尚未加载完成
- 当前页面不是动态详情页
- 评论区结构发生变化
- 其他浏览器扩展修改了页面结构
- 当前评论按钮处于 Shadow DOM 中，但该 Shadow DOM 不是 open 模式
- 页面样式主题改变了按钮颜色，导致识别失败

### 为什么已点赞数量和实际数量不完全一致？

脚本只能统计本次运行期间成功尝试处理的按钮数量，不一定等于页面中所有已点赞评论数量。

如果某些评论在脚本运行前已经点赞，脚本会跳过它们，但不一定计入“已点赞”统计。

### 为什么脚本会停止？

脚本会在连续多轮没有发现新候选按钮时自动停止，避免无限滚动和无意义扫描。

也可以手动点击“停止”中断运行。
